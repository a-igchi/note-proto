# 全体設計書

本ドキュメントではNote Appの概要について記述する。

## 名称

ユーザーが扱う最小単位のドキュメントは「**ノート**」と呼ぶ。これを唯一の正式呼称とし、他の語（pages / page / ページ / ドキュメント等）は UI・URL・ドキュメント上で利用しない。

| 文脈                                | 表記                                           |
| ----------------------------------- | ---------------------------------------------- |
| 日本語 UI（画面・ダイアログ・本文） | `ノート`                                       |
| 英語 / コード（識別子・型・URL）    | `note` / `Note` / `notes`（`/notes/:id` など） |
| 内部キャッシュキー・queryKey        | `["notes"]`                                    |

`apps/pages/` というワークスペース名は Cloudflare Pages デプロイ向けの内部名で、ユーザーには露出させない（`<title>` はすべて「ノート」に揃える）。

## 目的

これは文書管理アプリ。類似するものに以下のようなアプリ、サービスがある。

- Obsidian
- Notion
- Roam Research
- Logseq

基本的にこれらのアプリはドキュメントを階層によって管理する。階層によって管理する方法では、階層は親と子の１対１の関係を作るが、現実では１つのドキュメントを複数の階層に所属させたい場面がある。それを解決する手段として、バックリンクやタグによってグラフによってドキュメントを管理するものもあるが、これらが持つ階層による管理は複雑で、ユーザがドキュメントを管理する上での障壁になっている。

そこで、グラフによる管理を採用しつつも直観的にドキュメントを管理できるアプリを作ることを目的とする。
