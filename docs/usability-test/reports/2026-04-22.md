# ユーザビリティテスト結果 2026-04-22

## 試行シナリオ

1. 起動直後: 真っ白な画面に右下「+」のみ。アプリ名・オンボーディング一切なし。タブ名「pages」だけが手がかり。
2. 「+」をクリック → `/notes/new` に遷移。タイトル欄（日本語「タイトル」）と本文欄（英語「Please enter...」）。保存ボタン無し。戻るボタンのみ。
3. タイトルに「買い物メモ」、本文に「今日買うもの」を入力後、`* 牛乳` と打ったら Markdown が箇条書きに変換された（Milkdown/Notion風ブロックエディタ）。
4. リスト中に `/` を打ってもメニュー出ず。リスト外に出て `/` を打つとブロック挿入メニュー（全部英語）が出た。
5. 戻るボタンで一覧画面に戻ると、**ノートが大きな青い円として中央に単独で表示**。リスト画面ではなくグラフ/キャンバス画面だったと判明。
6. 2つ目のノート「TODO」（本文「仕事のタスク」）を作成して戻ると、**両ノードとも画面外**に配置され何も見えず「壊れた？」と一瞬思う。
7. ドラッグしてみたらパン可能と判明。ホイールで少し拡大できる。個別ノードは力学的シミュレーションで動く模様。
8. ノードを右クリック → 「リンクを追加」「削除」メニューが出現。リンク追加→検索ボックスで相手を選んで線で接続成功。
9. Ctrl+K でコマンドパレット的な検索バー出現（UIにヒント一切なし）。しかし「TODO」「todo」では該当無し、「買」では「買い物メモ」が出る。検索結果をクリックしても Enter を押しても**何も起きない**。
10. 既存ノートを再度開いてみると詳細画面にはリンク情報も削除ボタンも無し。
11. タイトル空・本文ありで戻ろうとすると「タイトルは必須です」で戻れなくなる（両方空だと戻れる）。
12. リロードしてもデータは残存（IndexedDB `knowledge-graph`）。

## 発見した課題

### [major] 新規ノート追加後、画面外に配置されてキャンバスが空に見える

- **再現**:
  1. `+` で1つ目のノートを作って戻る → 中央に表示される
  2. `+` で2つ目のノートを作って戻る → **両方とも画面外**
  3. `+` で3つ目のノートを追加するとまた画面外に混じる
- **期待していた挙動**: 新しく追加したノードは必ず見える位置に配置される。少なくとも全ノードが収まるように自動でズームアウト/センタリングされる。
- **実際の挙動**: viewport は 1280×720 なのに、TODO ノードの y 座標が -230、1つ目の新規ノードの y 座標が 872 など、平然とはみ出る。初見ユーザーは「ノートを作ったのに消えた」「アプリが壊れた」と判断する可能性が高い。
- **観測**: `./screenshots/2026-04-22/12-canvas-two-notes.png`（2つ作ったのに何も見えない）/ `./screenshots/2026-04-22/33-three-notes.png`（3つ目が画面外、上のノードもラベル見切れ）。getBoundingClientRect を eval して確認済み。
- **仮説**: d3-force などの力学シミュレーションに任せてそのまま初期位置を使っているだけで、**viewBox をノード群に合わせてフィットするロジックが無い**。`fitView()` 的な処理をノード追加後に走らせれば解消する。初見を壊す一番大きい原因なのでここを直すだけでアプリの印象が激変する。

### [major] Ctrl+K の検索パレットが事実上機能していない

- **再現**:
  1. Ctrl+K で検索バーを開く
  2. ノート名の一部（例: 「買」）を入れると候補「買い物メモ」が出る
  3. 候補をクリック、または Enter → **何も起きない**（URL 変化なし、ハイライトも消えない）
- **期待していた挙動**: 候補を選んだら該当ノートを開く、もしくはそのノードにキャンバスがフォーカス/ズームする。
- **実際の挙動**: クリックでも Enter でも反応せず、結果候補だけ表示されたまま詰まる。Esc で閉じるしかない。
- **観測**: `./screenshots/2026-04-22/27-after-search-click.png`（クリック後も URL は `/` のまま）/ `./screenshots/2026-04-22/28-search-enter.png`。
- **仮説**: 「リンクを追加」フローの検索UIを再利用しているが、検索側のクリックハンドラが実装されていない（リンク追加側にだけ onSelect が紐付いている）。UI が全く同一なだけに、動いてる方を見たユーザーほど不具合感が強い。

### [major] 検索が ASCII タイトルにも本文にもヒットしない

- **再現**:
  1. タイトル「TODO」のノートを作成（実在）
  2. Ctrl+K で「TODO」や「todo」を検索 → 「該当するノートがありません」
  3. タイトル「買い物メモ」・本文「仕事のタスク」のノート状態で「仕事」を検索 → 該当なし
  4. 「買」だけは引っかかる
- **期待していた挙動**: タイトルで大小文字区別なくヒット。本文もできれば検索対象。
- **実際の挙動**: ASCII は丸ごとスキップされる。本文はそもそも対象外。
- **観測**: `./screenshots/2026-04-22/23-search-todo.png` / `./screenshots/2026-04-22/24-search-lowercase.png` / `./screenshots/2026-04-22/26-search-body.png`。
- **仮説**: `title.includes(q)` ではなく、日本語向けの形態素/n-gram 前処理で ASCII がフィルタされている、あるいはインデクスに半角英数が入っていない。本文もインデックス対象外。`toLowerCase` した上で `title + body` に対する部分一致で十分だと思う（少なくとも初期実装としては）。

### [major] キャンバスにパン/ズームできることを示すUIが一切無い

- **再現**:
  1. ノートを2つ以上作る（上記課題のせいで画面外配置になる）
  2. ユーザーは空白の画面を見る
- **期待していた挙動**: 少なくとも「ドラッグで移動」「ホイールでズーム」のヒント、もしくは「全体表示」ボタン、ミニマップ、ズームコントロール、ヘルプアイコンのいずれか。
- **実際の挙動**: 何の手がかりも無く空白。たまたまドラッグを試したユーザーだけが進める。
- **観測**: `./screenshots/2026-04-22/12-canvas-two-notes.png`。UI要素は右下の「+」のみ。
- **仮説**: グラフキャンバスUIはそもそも初見ユーザーに厳しいので、初回だけのオンボーディングオーバーレイ、もしくは常設の「全体表示」ボタンとズームスライダーを右下か左下に用意すると救える。

### [minor] 言語表記が日英混在

- **再現**: どの画面を見ても混ざっている
  - タブ/ドキュメントタイトル: `pages`
  - URL: `/notes/new`、`/notes/:id`
  - 戻るボタン: `戻る`
  - タイトル欄placeholder: `タイトル`
  - 本文placeholder: `Please enter...`
  - スラッシュメニュー: `Text / List / Advanced / Heading 1 ...`
  - 検索: `ノートを検索...`、`該当するノートがありません`
  - 削除確認: `ノートを削除`、`このノートを削除しますか？この操作は取り消せません。`
  - コンテキストメニュー: `リンクを追加` / `削除`
- **期待していた挙動**: どちらかに統一（日本語UIなら `Please enter...` → `入力してください`、ブロック名も日本語訳）。
- **実際の挙動**: 混在したまま。
- **観測**: ほぼ全スクショに現れる。
- **仮説**: Milkdown/Crepe のデフォルトUIが英語のまま、周辺UIは自前で日本語化されている。ただアプリ全体の呼称すら「pages」「notes」で揺れているので、**まずどの名詞でノートを指すかをプロダクトとして決める**のが先。

### [minor] スラッシュコマンドの発動条件が一貫しない

- **再現**:
  1. 本文でリスト項目内にカーソルを置いて `/` を入れる → ただのスラッシュ文字として入る
  2. リストから抜けて段落で `/` を入れる → ブロック挿入メニューが出る
- **期待していた挙動**: どのブロック内でも `/` でメニューが出る（Notion/Obsidianはそう）。
- **実際の挙動**: リスト内では機能しない。
- **観測**: `./screenshots/2026-04-22/07-slash-menu.png`（リスト内でスラッシュだけ残る）/ `./screenshots/2026-04-22/09-slash-in-paragraph.png`（段落ではメニューが出る）。
- **仮説**: Milkdown/Crepe の設定で slash-plugin が paragraph ノード限定になっている可能性。リスト内も有効化できればコスト低く改善できるはず。

### [minor] 削除確認ダイアログに対象ノート名が出ない

- **再現**:
  1. 任意のノードを右クリック → 削除
  2. ダイアログに「ノートを削除 / このノートを削除しますか？この操作は取り消せません。」
- **期待していた挙動**: 「『TODO』を削除しますか？」のように対象名を出す。複数ノードがある状況で、右クリック位置とメニュー位置がズレてどれを削除しようとしているか不安になる。
- **実際の挙動**: 対象が「このノート」としか書かれていない。
- **観測**: `./screenshots/2026-04-22/21-after-delete.png`。
- **仮説**: 文言に `{title}` を埋めるだけで解消。

### [minor] 保存UIが無く自動保存の挙動がユーザーに伝わらない

- **再現**:
  1. ノートを編集しても「保存」「保存しました」「○秒前に自動保存」等の表示が一切無い
  2. 戻るボタンを押すと保存される（ようだ）
- **期待していた挙動**: 「自動保存済み」等のステータス、もしくは明示的な保存ボタン。少なくとも「戻る = 保存して戻る」であることが分かる文言。
- **実際の挙動**: 戻るボタンが破棄なのか保存なのかが判別できない。実際にはリロードすると残っているので保存はされている。
- **観測**: `./screenshots/2026-04-22/02-new-note.png` 以降、編集画面には一度も「保存」という文字が出ていない。
- **仮説**: ヘッダーに `保存済み` インジケータを置くか、戻るボタンのラベルを `← 保存して戻る` にするのが簡単。

### [minor] タイトル必須バリデーションが戻ろうとした瞬間まで隠れている

- **再現**:
  1. 本文だけ書いてタイトルを空のまま戻るボタンを押す
  2. 「タイトルは必須です」が出て戻れない
  3. 両方空の場合は何も言われず戻れる（未保存破棄扱い）
- **期待していた挙動**:
  - 必須であることを UI に最初から出す（ラベル `タイトル *` や placeholder を `タイトル（必須）` にするなど）
  - もしくは「タイトル未入力のまま戻ると破棄されます」を明示して戻れるようにする
- **実際の挙動**: 本文だけ書いた後で突然「戻れない」状態に陥る。破棄して離れたい時の逃げ道が無い。
- **観測**: `./screenshots/2026-04-22/32-after-body-only-back.png`。
- **仮説**: バリデーションを「保存時」ではなく「入力時 or UI 初期」に移すのが本来。もしくは戻るボタンを「破棄して戻る」「保存して戻る」に分ける。

### [minor] 詳細画面にバックリンクも削除ボタンも無い

- **再現**:
  1. リンクされた既存ノートをクリックして詳細を開く
  2. どこにも「このノートにリンクしているノート」「このノートがリンクしているノート」の表示が無い
  3. 削除動線も無い（削除はキャンバスの右クリックのみ）
- **期待していた挙動**: 知識グラフ系アプリなら詳細画面で backlinks / outbound links が一覧できるのが定番。削除も詳細画面から可能であるべき。
- **実際の挙動**: 詳細画面はただのタイトル+本文エディタで、グラフの存在を感じさせない。
- **観測**: `./screenshots/2026-04-22/34-edit-existing.png`。
- **仮説**: 「キャンバスに戻らないとそのノートとの関係が分からない」設計は知識グラフ系のUXとして弱い。小さくてもいいので詳細画面の下に「リンク」セクションを置くと一気に活きる。

### [polish] タブタイトルが固定で「pages」

- **再現**: 任意のノートを開いてもブラウザタブは `pages` のまま。
- **期待していた挙動**: 開いているノート名がタブに反映される（例: `買い物メモ - pages`）。
- **実際の挙動**: 常に `pages` 固定。
- **観測**: 全ページで確認。
- **仮説**: `document.title` を更新する処理が無いだけ。

### [polish] 内部名（notes）と外向け名（pages）が揺れている

- **再現**: URL は `/notes/:id`、タブは `pages`、UI中には「ノート」という言葉も出る。
- **期待していた挙動**: プロダクトで一つの名詞に統一。
- **観測**: URLとタブとコンテキストメニュー。

## 総評

**一言でいうと**: 知識グラフ（Roam/Obsidian系）を目指しているアプリだと思うが、**初見ユーザーがそこに気付けない段階で詰む**。

最も効くのは `[major] 初期フィッティング` と `[major] パン/ズームの手がかり不在` の2つ。この2つが直るだけで「空白画面で壊れたと思って離脱」するユーザーがかなり減る。次に `[major] Ctrl+K 検索が機能していない / ASCII に当たらない` が続く。これは検索を使ってみた人だけが踏むので体感的な優先度はその次。

細部では、**日英混在の放置**と**保存インジケータ不在**が、仕上がりの粗さとして一番目立つ。文言の統一は実装コストも低いのでブランド印象を上げるために早めに潰したい。

知識グラフアプリとしての核機能（ノート間リンク、グラフビュー）はちゃんと動いており、そこは好感が持てる。ただ**詳細画面にリンク情報が一切出ない**のが本アプリの「グラフ性」を体感しづらくしている。
